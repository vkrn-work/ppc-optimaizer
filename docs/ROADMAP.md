## Уровень 1 — Рекламная аналитика (MVP)

### 1.1 Инфраструктура и сбор данных ✅
- Схема БД: таблицы accounts, campaigns, ad_groups, keywords, keyword_stats
- Коллектор Яндекс Директ API v5: кампании, группы, ключи, статистика
- Коллектор Яндекс Метрика API: 12 срезов (сводка, кампании, ключи, устройства, регионы, посадочные, по дням, по часам, по дням недели, браузеры, тип пользователя, цели)
- Celery Beat: ежедневный сбор в 06:00 МСК
- Railway деплой: бэкенд + фронтенд + воркер

### 1.2 Анализ и предложения ✅
- CR-анализ: 5 типов проблем (low_position, traffic_drop, zero_ctr, low_ctr, click_position_gap)
- Генерация предложений из анализа
- Трекинг гипотез: 7-дневный мониторинг после применения изменения
- Вердикт гипотезы: confirmed / rejected / neutral / insufficient

### 1.3 Фронтенд — 11 страниц ✅
- Main Board: KPI с дельтами, спарклайны, проблемы и точки роста
- По кампаниям: кампании и ключевые слова с сортировкой
- Ставки: все кампании, фильтр по группам, CTR/CPC, рекомендации, калькулятор
- Корректировки: данные Метрики по устройствам, регионам, часам, дням недели
- Новые ключи / Минуса: поисковые фразы с коммерческим скорингом
- Предложения: проблемы и точки роста из анализа
- Гипотезы: создание и трекинг
- Кабинеты: управление OAuth-токенами и настройками
- Правила: просмотр правил анализа
- Диагностика: статус системы

### 1.4 Технические исправления v1.1.0 ✅
- Исправлен баг is_active для кампаний (State vs Status в API Директа)
- Конвертация AvgEffectiveBid из микрорублей
- Правильный расчёт CPC и CTR через суммы
- Параметр days для ретроспективного сбора (до 90 дней)
- Новый эндпоинт /ad-groups
- Время МСК в топбаре
- Переключение периода работает на всех страницах

## Уровень 2 — CRM-интеграция

### 2.1 Ручная загрузка из 1С (CSV) 🔲
**Цель:** связать рекламный клик с квалификацией лида без API.
- Формат CSV: lead_id, created_at, utm_source, utm_campaign, utm_term, roistat_id, client_id, status, status_updated
- Загрузка через UI (drag & drop или кнопка)
- Матчинг клик → заявка → статус по utm_term + client_id
- Появятся метрики: MQL, SQL, CPL, CPQL, CR клик→заявка, CR заявка→SQL

### 2.2 API 1С в реальном времени 🔲
**Цель:** заменить ручную выгрузку на автоматический вебхук.
- REST API или вебхуки из 1С
- Таблица leads: account_id, keyword_id, campaign_id, status, utm_term, created_at
- Автоматическое обновление воронки при изменении статуса лида

### 2.3 Дашборд CRM-блок 🔲
- Блок "Результат" на Main Board: заявки, SQL, КП, сделки
- CR по каждому этапу воронки
- CPQL по кампаниям и кластерам
- Сравнение периодов

## Уровень 3 — Автоприменение изменений

### 3.1 Директ API write-доступ 🔲
- Метод bids.set: изменение ставки ключевого слова
- Метод adGroups.update: добавление минус-слов
- Метод campaigns.update: изменение бюджета
- Перевод на стратегию CPA: изменение настроек кампании

### 3.2 Защита от ошибок 🔲
- Лимит: не более 20% ключей кампании за один запуск
- Автооткат: если через 48 часов CR упал > 30% — создать предложение об откате
- Крупные изменения (бюджет > 50%, отключение группы) — всегда требуют аппрува

### 3.3 UI аппрува 🔲
- Список предложений с кнопками Аппрув / Отклонить / Изменить значение
- После аппрува — автоматическое применение через API
- Лог всех применённых изменений

## Модуль Б — Собственная сквозная аналитика

### Б.1 Замена Roistat 🔲
**Цель:** убрать абонентскую плату ~30–80 тыс. ₽/мес.
- JS-скрипт на сайте: фиксирует UTM, реферер, устройство, страницы
- При отправке формы — запись в таблицу leads с visit_id
- Коллтрекинг: звонки с привязкой к источнику и кампании
- clientID Метрики как сквозной идентификатор
- Переход прозрачен: таблица leads та же, меняется только источник

## Модуль В — AI-ассистент

### В.1 LLM с доступом к БД 🔲
**Цель:** директолог работает с данными на естественном языке.
- Tool use: query_db (read-only SQL), get_campaign_stat, get_hypothesis, create_hypothesis, get_cluster_cr
