# Бизнес-контекст и методология

> Этот документ описывает *зачем* построена система и *как* она должна думать.
> Обязателен к прочтению перед работой с кодом анализатора и генератора предложений.

---

## 1. Контекст бизнеса

| Параметр | Значение |
|---|---|
| Ниша | Поставки импортного металлопроката (трубы, листы, полосы по стандартам ASTM, DIN, EN и др.) |
| Сегмент | B2B — закупщики, снабженцы, технические специалисты промышленных предприятий |
| Реклама | Поисковые кампании Яндекс Директ по ключевым словам |
| Регион | Российская Федерация |
| CRM | Битрикс24 (задачи директолога) + 1С (воронка продаж) |
| Сквозная аналитика | Roistat → собственная система (целевая) |
| Цель | 200 SQL в месяц при CPQL ≤ 6 000 ₽ |

**Проблемы, которые решает система:**
- Рутинный сбор статистики занимает 5–10 часов в неделю
- Решения по ставкам принимаются редко и без статистической базы
- Длинный хвост семантики (низкочастотные ключи по стандартам) не оптимизируется
- Нет связки между рекламным кликом и квалификацией лида в CRM
- Гипотезы не документируются и не проверяются системно

## 2. Специфика B2B-интента в нише металлопроката

Понимание интента критично для анализатора — нельзя слепо оптимизировать по CTR.

### Типы поисковых запросов

**Снабженец с ТЗ** — ищет конкретную спецификацию с размерами и маркой. Конверсия максимальная. Признаки:
- Конкретные размеры: `12.7×1.65`, `114.3×6`, `8мм`
- Марка стали: `TP316L`, `AISI 316`, `S690QL`, `30MnB5`
- Стандарт + марка: `ASTM A269 TP316L`, `EN 10149 S700MC`
- Производственные аббревиатуры: `SMLS`, `BE`, `SCH`
- Сертификаты: `РМРС`, `NACE`, `dual grade`

**Снабженец без ТЗ** — ищет по стандарту, потом уточняет. Нужна экспертная посадочная.

**Проектировщик/инженер** — ищет технические данные. Информационный интент, заявки редко. Минусовать или отдельный лендинг.

### Признаки нецелевого запроса (в минус)
- `стандарт ASTM A269` — ищут документ, не трубу
- `что такое`, `описание`, `характеристики` без `купить`
- Аббревиатуры без продуктового слова: просто `ASTM A269`
- `оформить сертификат РМРС` — ищут услугу, не продукт

### Сезонность
- **Февраль** — пик: закрытие квартальных заявок, конкуренция максимальная
- **Март начало** — спад после февраля, ставки снижать не нужно
- **Апрель–май, сентябрь–октябрь** — новый пик: весенние и осенние тендеры
- **Лето и январь** — спад, оптимальное время для тестов новой семантики

## 3. Структура семантики

Семантика организована в трёхуровневую иерархию:

| Уровень | Пример | Описание |
|---|---|---|
| Ключевое слово | `труба astm a333 gr.6 купить` | Атомарная единица |
| Кластер | Все ключи по ASTM A333 | Группа ключей одного товара/стандарта |
| Товарное направление | Все трубы по ASTM-стандартам | Группа кластеров одной ниши |

**Принцип кластеризации:** общий стандарт или марка материала.

**Правило:** один интент — одна группа. Коммерческие и информационные запросы по одному стандарту не должны быть в одной группе.

**"Золотые фразы"** — запросы с конкретными размерами и маркой (`труба 12.7×1.65 smls astm a269 tp316l`). Самые квалифицированные лиды. Их нет в подборщике ключей — приходят только из отчёта по поисковым фразам. Собирать регулярно и добавлять как точные ключи.

## 4. Логика CR-анализа и биддинга

### Двухуровневая модель ставок

**Уровень 1 — ставка кластера (базовая)**
```
Ставка_кластера = целевой_CPL × CR_кластера
```
Применяется ко всем ключам кластера с менее 30 кликами.
Пример: CPL цель 1500₽, CR кластера ASTM A333 = 16% → базовая ставка = 240₽

**Уровень 2 — индивидуальный коэффициент ключа**
```
Ставка_ключа = Ставка_кластера × (CR_ключа / CR_кластера)
```
Применяется только если у ключа ≥ 30 кликов (статистически значимо).

### Маршрутизация ключей по CR

| Условие | Кликов | Действие | Модель оплаты |
|---|---|---|---|
| CR > 15% | ≥ 30 | Поднять ставку | CPC |
| CR 5–15% | ≥ 30 | Держать ставку | CPC |
| CR 1–5% | ≥ 100 | Перевести на CPA | CPA |
| CR < 1% | ≥ 100 | Минус-слова / отключить | — |
| Любой CR | < 30 | Накапливать данные | Ставка кластера |

### Статистическая значимость
- Ключ: минимум **30 кликов** для вывода о CR
- Группа: минимум **50 кликов**
- Кампания: минимум **100 кликов**

До порога — использовать CR кластера как базу, не CR отдельного ключа.

## 5. Диагностическое дерево: причины ухудшения

### Причина A: Снижение ставок (самая частая)
**Диагноз подтверждён если одновременно:**
1. Объём трафика (спрос) в системе стабилен
2. Клики резко упали
3. Средняя ставка снизилась по сравнению с прошлым периодом
4. Позиция показа ухудшилась

В ЕПК при стратегии «ручные ставки» алгоритм **всё равно автоматически снижает ставки** ключей без конверсий. Это системное поведение, не баг. Решение: перевод на ТГК (текстово-графическую кампанию) с фиксированными ставками.

### Причина B: Каннибализация
Несколько кампаний с одной семантикой (CPC + ОЗК + конверсионная) конкурируют на одном аукционе → CPC растёт, бюджет тратится быстрее, кликов меньше.
Решение: оставить одну стратегию.

### Причина C: Информационный трафик в коммерческих группах
Группа получает клики, CR = 0% при 30+ кликах. В поисковых фразах нет коммерческих сигналов.
Решение: минус-слова или перевод на ОЗК.

### Причина D: Несоответствие посадочной запросу
Трафик целевой, но CR упал. Посадочная не упоминает стандарт/марку по которой идёт трафик.

**Чеклист посадочной:**
- Заголовок содержит стандарт или марку из топ-групп
- В ассортиментном блоке перечислены стандарты и марки
- Форма видна без скролла
- Нет избыточных обязательных полей (ИНН, реквизиты) на первом контакте
- Есть конкретный оффер с дедлайном
- Есть WhatsApp / быстрый контакт

## 6. Формат задач и гипотез

Каждая задача по оптимизации фиксируется в структуре:

| Поле | Описание |
|---|---|
| Объект | Группа / ключ / кампания / посадочная |
| Наблюдение | Что видим в данных |
| Гипотеза | Предполагаемая причина |
| Действие | Конкретное изменение |
| Ожидаемый результат | Измеримый эффект через N дней |
| Метрика проверки | Как проверить |
| Статус | Открыта / В работе / Подтверждена / Отклонена |

**Приоритизация задач:**

| Уровень | Критерий | Срок |
|---|---|---|
| 🔴 Сегодня | Прямая потеря денег: ставки упали по флагманам | 24 часа |
| 🟡 Эта неделя | Минус-слова, стратегия, посадочная | 3–5 дней |
| 🔵 До конца месяца | Новые группы, кластеры, кампании | 1–4 недели |
| 🟢 Масштабирование | Новые направления, автоматизация | 1–3 месяца |

## 7. Роль директолога после внедрения системы

С внедрением системы роль смещается от операционной к управленческой:

| Задача | До системы | После системы |
|---|---|---|
| Сбор статистики | 5–10 ч/нед вручную | Автоматически, 0 ч |
| Формирование отчёта | 3–5 ч/нед | Автоматически, 0 ч |
| Корректировка ставок | 2–4 ч/нед | Аппрув предложений, 30 мин |
| Мониторинг кампаний | Ежедневно, 1 ч | Алерты при аномалиях |
| Семантика и объявления | 10–20 ч на запуск | LLM генерирует, валидация 1–2 ч |
| Количество направлений | 3–5 качественно | 15–25 качественно |

## 8. Пример реального анализа (кампания 704449682)

Этот пример показывает как должна работать система на практике.

**Ситуация:** Листовой прокат ЕПК. Февраль: 125 кликов, 8 заявок, CPL 1 244₽. Март: 76 кликов (−39%), 2 заявки (−75%), CPL 4 317₽ (+3.5×).

**Диагноз:** ЕПК автоматически снизила ставки ключей без конверсий. Три ключа-лидера по S690QL, S700MC, S315MC потеряли ставки и выпали из показов. Объём трафика в системе сохранялся (~100–115 ед.) — спрос был, участия в аукционе не было.

**Задачи с приоритетами:**
- 🔴 Восстановить ставки S690QL (CR 27% в феврале, 3 заявки + 1 БП)
- 🔴 Восстановить ставки S700MC (25 кликов, 1 заявка + 1 БП в феврале)
- 🔴 Восстановить 13 ключей с обнулёнными ставками
- 🟡 Убрать дубли EN 10088 (4 ключа конкурируют друг с другом)
- 🔵 Оценить перевод с ЕПК на ТГК

**Ожидаемый результат после 🔴-задач:** клики 76/нед → 110–130/нед, заявки 2 → 6–8/нед, CPL 4 317₽ → 1 200–1 500₽.

Именно такого уровня детализации должны достигать предложения системы.
